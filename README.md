# Mojo 心智模型

理解 **Value Semantic（值语义）** 的关键不是"复制"，而是：

> **变量 = 值本身，而不是对象的引用。**

很多人把值语义理解成 **"赋值会 copy"**，其实这只是表现，不是本质。

真正的本质是：

> **变量拥有完整的数据，而不是指向数据。**

------------------------------------------------------------------------

# 一、值语义的本质

值语义的核心规则只有一个：

    变量 = 数据

而不是：

    变量 -> 数据

------------------------------------------------------------------------

## 值语义（Value Semantic）

    a = 10
    b = a

内存：

    a: 10
    b: 10

两个**独立数据**。

修改：

    b = 20

结果：

    a: 10
    b: 20

互不影响。

------------------------------------------------------------------------

## 引用语义（Reference Semantic）

例如 Python：

    a = [1,2,3]
    b = a

内存：

    a ----\
           -> [1,2,3]
    b ----/

修改：

    b.append(4)

结果：

    a = [1,2,3,4]
    b = [1,2,3,4]

因为共享对象。

------------------------------------------------------------------------

# 二、值语义的真正意义

值语义其实解决的是一个问题：

> **谁拥有数据？**

在值语义中：

    每个变量都拥有自己的数据

所以：

    ownership = clear

这也是 Rust / Mojo / Swift 都采用值语义的重要原因。

------------------------------------------------------------------------

# 三、值语义不是一定会 copy

值语义：

    语义 = copy

但：

    实现 ≠ 必须 copy

编译器可以优化。

### Copy Elision

例如：

    fn make() -> Int:
        var x = 10
        return x

编译器可能直接返回寄存器，而不是复制。

### Move Optimization

如果变量不再使用，编译器可以 **move** 而不是 **copy**。

Rust 和 Mojo 都会做这种优化。

------------------------------------------------------------------------

# 四、值语义的真正抽象

值语义可以理解为：

    Pure Data Model

数据只有内容，没有：

    identity
    alias
    shared mutation

例如：

    10 == 10

两个 10 没有身份区别。

而对象模型中：

    object1 != object2

即使内容一样。

------------------------------------------------------------------------

# 五、值语义带来的优势

## 1 可预测

函数参数不会被外部偷偷修改。

## 2 并发安全

值语义避免共享可变状态，更容易并发。

## 3 简化思维模型

程序员只需要思考 **数据流**，而不是复杂的对象关系。

------------------------------------------------------------------------

# 六、值语义 vs 引用语义

  特性       值语义       引用语义
  ---------- ------------ ---------------
  数据       独立         共享
  修改       不影响其他   影响所有引用
  并发       更安全       容易出现 race
  identity   没有         有
  alias      不存在       常见

------------------------------------------------------------------------

# 七、Mojo 的值语义

Mojo 是 **Value First Language**。

例如：

``` mojo
struct Vec2:
    var x: Float
    var y: Float

var a = Vec2(1,2)
var b = a
```

内存：

    a: {1,2}
    b: {1,2}

两个完全独立的值。

------------------------------------------------------------------------

# 八、值语义最深层的本质

对象模型：

    variable -> object -> state

值模型：

    variable -> state

少了一层 **object identity**。

这就是值语义更简单、更安全的原因。

------------------------------------------------------------------------

# 九、一句话总结

> **值语义 = 没有 alias 的数据模型**

也就是说：

    两个变量不会共享同一份可变数据

除非你显式使用 **引用或指针**。
